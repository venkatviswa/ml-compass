How the deterministic engine chooses a model ladder

never one winning algorithm — a defensible baseline and a sequence of families to test

flowchart TD
    A["Dataset profile
+ business context"] --> T{"Target
selected?"}
    T -->|No| U{"Analysis goal?"}
    U -->|group records| UC["Clustering
KMeans first;
DBSCAN/HDBSCAN for irregular groups"]
    U -->|compress / visualize| UR["Dimensionality
PCA to compress;
UMAP / t-SNE to visualize"]
    U -->|find rare records| UA["Anomaly detection
Isolation Forest first; One-Class SVM
or autoencoder as needed"]
    T -->|Yes| M{"Data
modality?"}
    M -->|text| TX["Text ladder
TF-IDF + Logistic / Naive Bayes
→ transformer if the baseline falls short"]
    M -->|images| IM["Image ladder
small CNN sanity check
→ pretrained CNN / vision transformer"]
    M -->|tabular| K{"Target
type?"}
    K -->|continuous| RG["Regression ladder
mean baseline → Linear Regression
→ Random Forest → LightGBM / XGBoost"]
    K -->|categorical| CL["Classification ladder
majority baseline → Logistic Regression
→ Random Forest → boosting (CatBoost if high-card)"]
    K -->|ordered values| OR["Ordinal ladder
mean baseline → ordinal logistic
→ Frank–Hall cumulative classifiers"]
